# Classifier Agent

**Role:** Identify non-technical companies in US/UK/Europe that want AI automation but lack the technical team to implement it — then generate a custom proposal.

**Runtime:** Python (`classifier_agent.py`) — 3 Claude agents run in parallel per company  
**Model:** `claude-opus-4-8` × 3 agents (parallel async)  
**Output:** Obsidian notes + local JSON cache  
**Triggered by:** `python cli.py classify`

---

## Purpose

Two parallel revenue tracks:

| Track | Target | Action |
|---|---|---|
| **Track A** | Technical companies (Series A/B, Rust shops, AI infra) | Job application pipeline |
| **Track B** | Non-technical companies (wealth families, law firms, property, logistics) | AI automation proposal + implementation |

Track B is the classifier's primary focus. These companies have:
- **Money** — professional services, asset management, family offices generate large revenues
- **Pain** — manual Excel workflows, paper-based processes, repetitive email/reporting
- **No tech team** — they have heard of AI but don't know where to start
- **Decision power** — CEO or COO can say yes in one meeting

## Architecture

Three specialized Claude agents run simultaneously per company. A Decision Router picks the track.

```
Company (from DB or manual input)
         │
         ├── [parallel data fetch]
         │     ├── WebReveal   → tech stack (low count = non-technical signal)
         │     └── Website     → AI mentions, careers page, text signals
         │
         ├── [3 Claude claude-opus-4-8 agents, parallel]
         │     ├── Agent 1: Tech Classifier
         │     │     "Is this company technical or non-technical?"
         │     │     Output: tech_maturity 0-10, sector, country, size
         │     │
         │     ├── Agent 2: AI Hunger Detector
         │     │     "Do they want/need AI automation? What are their pain points?"
         │     │     Output: hunger_score 0-10, pain_points, manual_processes
         │     │
         │     └── Agent 3: Proposal Generator (Track B only)
         │           "What specific AI solution do we propose?"
         │           Output: solution, deliverables, pricing, email draft
         │
         └── [Decision Router]
               ├── TRACK A → technical, existing job pipeline
               └── TRACK B → non-technical + hunger ≥ 6 → Obsidian proposal note
```

## Three Agents

### Agent 1 — Tech Classifier

**Question:** Is this a technical company or a non-technical company?

**Evidence it looks for:**
- Tech stack detected by WebReveal (WordPress/Wix/HubSpot = non-technical signal)
- Careers page: are they hiring engineers or accountants?
- Company type: estate manager ≠ software company
- GitHub org: exists? Active?

**Output:**
```json
{
  "is_technical": false,
  "tech_maturity": 2,
  "company_size": "small (10-50)",
  "sector": "wealth management",
  "country": "United Kingdom",
  "non_tech_signal": "WordPress site, no GitHub, job postings for PAs and analysts only",
  "one_liner": "London family office managing £500M in private equity and real estate",
  "confidence": "high"
}
```

**Threshold for Track B:** `is_technical = false` OR `tech_maturity ≤ 4`

---

### Agent 2 — AI Hunger Detector

**Question:** Does this company need AI? How urgently? What's their pain?

**Sector-specific pain points it understands:**
- **Wealth:** Automated portfolio reports generated by hand monthly
- **Legal:** Contract review, NDA analysis, client intake forms — all manual
- **Real estate:** Property comparison, tenant screening, maintenance logs in Excel
- **Logistics:** Route emails, vendor updates, customs paperwork — still faxed
- **Recruitment:** CV screening, interview scheduling — one person manually reviewing

**Output:**
```json
{
  "hunger_score": 8,
  "why_they_want_ai": "Monthly investor reports take 3 analysts 2 days each to compile",
  "pain_points": ["Manual Excel reporting", "Email client updates one by one", "No CRM integration"],
  "manual_processes": ["Portfolio summary emails", "NAV calculation", "Meeting notes"],
  "decision_maker_signal": "CEO or COO — investment decisions flow through top 2 people",
  "urgency": "high",
  "budget_signal": "AUM indicates >£1M operational budget; partners bill >£500/hr"
}
```

**Threshold for Track B:** `hunger_score ≥ 6`

---

### Agent 3 — Proposal Generator (Track B only)

**Question:** What specific AI system do we build for this company? What do we charge?

**What it produces:**
- Concrete solution name (not generic)
- 4-6 specific deliverables
- Timeline (MVP → full system)
- Realistic price range for their geography and size
- Who to contact and how
- Ready-to-send email draft

**Output:**
```json
{
  "solution_title": "AI Operations Suite for Family Offices",
  "solution_description": "Automated investor reporting, portfolio data extraction, and client email system. Reduces monthly reporting from 3 days to 3 hours. Runs privately — no data leaves your infrastructure.",
  "deliverables": [
    "Portfolio report generator (Claude API + your data sources)",
    "Investor email draft system (personalized by portfolio performance)",
    "Document extraction pipeline (NAV statements, bank feeds)",
    "Simple web dashboard for your team (no tech knowledge needed)",
    "Private deployment on your own server or cloud account"
  ],
  "tech_stack_we_use": ["Claude API", "Python", "SQLite", "PDF extraction", "SMTP automation"],
  "timeline": "4 weeks MVP, 8 weeks full system",
  "estimated_value": "£18,000–28,000 setup + £2,500/mo maintenance",
  "pricing_model": "project",
  "target_contact_role": "CEO or Operations Director",
  "email_subject": "2 days of analyst time → 3 hours",
  "email_draft": "At firms like yours, monthly investor packs take your analysts 2–3 days to compile. We build AI systems that do it in 3 hours — automatically pulling from your data sources, generating report drafts, and personalizing client emails. No data leaves your control. Would you be open to a 20-minute call to see if this fits your workflow?"
}
```

## Decision Logic

| tech_maturity | hunger_score | Track |
|---|---|---|
| ≤ 4 | ≥ 6 | **TRACK B** — non-technical, AI-hungry → proposal |
| ≤ 4 | < 6 | Skip — non-technical but no clear AI need |
| ≥ 5 | any | **TRACK A** — technical company → job pipeline |

## Obsidian Output

All results auto-written to local Obsidian vault:

```
Obsidian Vault/
├── Prospects/
│   ├── _Dashboard.md              ← auto-updated index (all companies)
│   ├── TrackA_Jobs/
│   │   └── Zed Industries.md      ← job application note
│   └── TrackB_Proposals/
│       └── Vermeer Capital.md     ← proposal + email draft
└── Classifier/
    └── _Classifier_Log.md         ← all runs logged
```

Each **Track B note** contains:
- Company profile (sector, country, size, tech maturity)
- AI hunger assessment + pain points
- Full proposal (solution, deliverables, timeline, pricing)
- Ready-to-send email draft
- Outreach checklist (email → follow-up × 2 → proposal → contract)

## Running the Classifier

```bash
# Classify all new prospects in DB (from research/run commands)
python cli.py classify --mode db --limit 20

# Classify a single company manually
python cli.py classify --mode manual \
  --company "Vermeer Capital" \
  --website "vermeercap.com" \
  --location "London, UK" \
  --sector "wealth management"

# Preview without writing to Obsidian
python cli.py classify --mode db --limit 10 --dry-run

# Higher concurrency for large batches
python cli.py classify --mode db --limit 50 --concurrency 5
```

## Data Sources for Finding Track B Companies

### US/UK Companies House (free)
```bash
# UK Companies House API
GET https://api.companieshouse.gov.uk/search/companies?q=family+office&items_per_page=20
# Free, no key needed. Filter by SIC code:
# 64205 = activities of financial holding companies
# 64301 = activities of investment trusts
# 68209 = other letting and operating of own or leased real estate
```

### LinkedIn (manual research)
- Search: `"family office" OR "wealth management" site:linkedin.com/company location:UK`
- Look for companies with <50 employees in finance/property/legal

### Local directories
- **UK:** Companies House, FCA register, Chambers of Commerce
- **US:** Dun & Bradstreet, IRS nonprofit/LLC databases
- **Europe:** national business registries (Handelsregister DE, KVK NL, SIREN FR)

## Full Pipeline Integration

```
# Step 1: Discover prospects (any source)
python cli.py run --query "wealth management" --location "London" --mode yc

# Step 2: Classify (splits into Track A + Track B)
python cli.py classify --mode db --limit 20

# Step 3a: Track A → apply + outreach
python cli.py cv --jd <jd>.txt --company "Zed" --role "Rust Engineer"
python cli.py email --company "Zed" --contact "..." --signal "..."

# Step 3b: Track B → send proposal email (use Obsidian draft)
# Open TrackB_Proposals/<Company>.md in Obsidian
# Copy email draft → send from Gmail/Outlook
# Tick checklist as you progress

# Step 4: Track in Obsidian
python cli.py dashboard   # DB-level status
# Obsidian: update status field in note frontmatter
```
